在 **Arch Linux** 上离线部署 Docker，与 CentOS 的 `repotrack + createrepo` 或 Rocky 的 `reposync` 思路类似，但 Arch 使用的是 **pacman 包管理器**，没有 rpm 仓库体系，因此最佳实践是：

> 在有外网的 Arch Linux 机器上同步 Docker 及其全部依赖包 → 拷贝到移动硬盘 → 内网机器构建本地仓库 → 使用 pacman 安装。

这是生产环境最稳定、可维护的方法。

---

# 方案一：构建 Arch Linux 本地仓库（推荐）

## 架构图

```text
互联网机器
    │
    ├── 下载 docker + 依赖
    │
    ├── repo-add 创建仓库
    │
    └── 移动硬盘
             │
             ▼

内网 Arch Linux
    │
    ├── 挂载移动硬盘
    │
    ├── 配置 pacman 本地仓库
    │
    └── pacman 安装 Docker
```

---

# 第一步：外网机器下载 Docker 全部依赖

先查看依赖：

```bash
pactree -u docker
```

例如：

```text
docker
├─containerd
├─runc
├─iptables
├─iproute2
├─libseccomp
├─systemd
└─...
```

---

## 创建下载目录

```bash
mkdir -p /data/docker-repo
cd /data/docker-repo
```

---

## 下载 Docker 及依赖

推荐使用：

```bash
pacman -Sw \
docker \
containerd \
runc \
iptables \
iproute2 \
libseccomp \
--cachedir .
```

只下载不安装。

---

## 自动下载全部依赖（推荐）

安装：

```bash
sudo pacman -S pacman-contrib
```

执行：

```bash
pactree -u docker
```

导出：

```bash
pactree -u docker | sort -u > pkglist.txt
```

批量下载：

```bash
pacman -Sw $(cat pkglist.txt) \
--cachedir .
```

---

# 第二步：创建离线仓库

Arch 官方提供：

```bash
repo-add
```

生成仓库索引：

```bash
repo-add docker-local.db.tar.gz *.pkg.tar.zst
```

生成：

```text
docker-local.db.tar.gz
docker-local.files.tar.gz
```

目录结构：

```text
docker-repo/
├── docker-28.x.pkg.tar.zst
├── containerd.pkg.tar.zst
├── runc.pkg.tar.zst
├── ...
├── docker-local.db.tar.gz
└── docker-local.files.tar.gz
```

---

# 第三步：复制到移动硬盘

例如：

```bash
rsync -av docker-repo /mnt/usb/
```

---

# 第四步：内网 Arch Linux 配置本地仓库

复制：

```bash
mkdir -p /opt/localrepo

cp -r /mnt/usb/docker-repo/* \
      /opt/localrepo/
```

---

修改：

```bash
vim /etc/pacman.conf
```

添加：

```ini
[docker-local]
SigLevel = Optional TrustAll
Server = file:///opt/localrepo
```

---

刷新仓库：

```bash
pacman -Sy
```

查看：

```bash
pacman -Sl docker-local
```

---

# 第五步：安装 Docker

```bash
pacman -S docker
```

pacman 会自动解析依赖。

---

# 第六步：启动 Docker

```bash
systemctl enable docker
systemctl start docker
```

验证：

```bash
docker version
```

---

# 方案二：直接复制 pacman 缓存（简单）

适合少量机器。

外网机器：

```bash
pacman -Sw docker
```

缓存目录：

```bash
/var/cache/pacman/pkg
```

复制：

```bash
docker*.pkg.tar.zst
containerd*.pkg.tar.zst
runc*.pkg.tar.zst
...
```

---

内网机器：

```bash
pacman -U *.pkg.tar.zst
```

例如：

```bash
pacman -U \
containerd.pkg.tar.zst \
runc.pkg.tar.zst \
docker.pkg.tar.zst
```

缺点：

* 必须手工排序依赖
* 后续升级困难

生产不推荐。

# 方案三：建立企业级 Arch 镜像仓库（推荐大规模集群）

如果内网有几十台以上机器，建议搭建：

```text
Nginx
 + Arch Repo
 + Docker Repo
 + 自研软件 Repo
```

目录：

```text
/repo
 ├── core
 ├── extra
 ├── community
 └── docker-local
```

同步官方仓库：

```bash
rsync \
rsync://mirrors.kernel.org/archlinux \
/repo/archlinux
```

然后：

```nginx
server {
    listen 80;

    location /archlinux {
        root /repo;
        autoindex on;
    }
}
```

客户端：

```ini
[core]
Server = http://repo-server/archlinux/core/os/x86_64

[extra]
Server = http://repo-server/archlinux/extra/os/x86_64

[docker-local]
Server = http://repo-server/docker-local
```

这样就类似：

* CentOS Repo Server
* Rocky Repo Server
* Debian Apt Mirror

企业环境维护最方便。

# 企业级最佳实践（推荐）

如果你的目标是：

* CentOS 7
* Rocky Linux 9
* Debian 13
* Arch Linux
* Alpine Linux
* OpenSUSE

统一离线部署 Docker

建议建立一个移动硬盘目录：

```text
offline-repo/
├── centos7/
├── rocky9/
├── debian13/
├── archlinux/
├── alpine/
├── opensuse/
└── docker-images/
```

其中：

```text
docker-images/
├── registry.tar
├── mysql.tar
├── redis.tar
├── nginx.tar
└── ...
```

同时提前导出镜像：

```bash
docker pull nginx:latest

docker save -o nginx.tar nginx:latest
```

内网：

```bash
docker load -i nginx.tar
```

这样不仅能离线安装 Docker，还能离线部署完整容器平台。

如果后面你的内网环境包含 **Arch Linux + Rocky Linux 9 + Debian 13 混合集群**，我还可以给你一套企业级离线仓库建设方案，包括：

* Arch（repo-add）
* Debian（apt-mirror + reprepro）
* Rocky（reposync + createrepo）
* OpenSUSE（zypper mirror）
* Alpine（apk index）
* Docker 镜像私服（Harbor + Docker Registry）

实现完全断网环境下的一键部署与统一运维。
