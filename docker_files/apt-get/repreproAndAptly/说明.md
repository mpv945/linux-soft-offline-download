# Aptly vs reprepro

### reprepro

特点：

```text
简单
轻量
稳定
占用小
```

适合：

```text
Docker
Kubernetes
Harbor
固定软件包
```

企业内网最常见。

---

### Aptly

特点：

```text
镜像同步
快照
版本冻结
回滚
发布
S3
```

适合：

```text
金融
军工
政企
离线环境
```

支持：

```bash
aptly mirror
aptly snapshot
aptly publish
```

类似：

```text
reposync
↓
snapshot
↓
createrepo
```

功能远强于 reprepro。([aptly.info][1])

先说一个容易混淆的地方：

## reprepro 不是下载器

很多人把 reprepro 理解成：

```bash
reprepro = apt-get download
```

实际上不是。

`reprepro` 的职责是：

```text
导入 .deb
↓
生成 APT 仓库元数据
↓
发布仓库
```

类似于：

```text
RPM:
createrepo

DEB:
reprepro
```

---

# 1 reprepro -b 能一次导入多个软件吗？

当然可以。

例如：

```bash
reprepro -b /opt/repo includedeb trixie *.deb
```

一次导入目录所有包。

例如：

```bash
reprepro -b /opt/repo includedeb trixie \
    containerd.io_*.deb

reprepro -b /opt/repo includedeb trixie \
    docker-ce-cli_*.deb

reprepro -b /opt/repo includedeb trixie \
    docker-ce_*.deb

reprepro -b /opt/repo includedeb trixie \
    docker-buildx-plugin_*.deb

reprepro -b /opt/repo includedeb trixie \
    docker-compose-plugin_*.deb
```

或者：

```bash
find debs -name "*.deb" \
    -exec reprepro -b /opt/repo includedeb trixie {} \;
```

---

# 2 reprepro 能自动下载依赖吗？

不能。

例如：

```bash
reprepro includedeb docker-ce.deb
```

如果缺：

```text
containerd.io
docker-ce-cli
```

reprepro 不会帮你下载。

因为它不是：

```text
apt
apt-get
aptitude
```

---

# 3 如何下载依赖？

应该先：

```bash
apt-get install \
    --download-only \
    docker-ce
```

或者：

```bash
apt download
```

或者：

```bash
apt-rdepends
```

下载完成后：

```bash
reprepro includedeb
```

导入仓库。

---

# 4 如何只保留最新版本？

reprepro 默认支持。

例如仓库里已有：

```text
docker-ce 28.3.0
docker-ce 28.3.1
docker-ce 28.3.2
```

执行：

```bash
reprepro remove trixie docker-ce
```

删除旧版本。

或者配置：

```text
conf/options
```

```text
keepunusednewfiles no
```

---

查看版本：

```bash
reprepro -b /opt/repo list trixie docker-ce
```

输出：

```text
trixie|main|amd64: docker-ce 28.3.2
```

---

# 5 reprepro 能指定版本吗？

可以。

例如：

```bash
docker-ce_28.3.3-1~debian.13~trixie_amd64.deb
```

导入：

```bash
reprepro -b /opt/repo includedeb trixie \
    docker-ce_28.3.3-1~debian.13~trixie_amd64.deb
```

APT客户端安装：

```bash
apt install docker-ce=5:28.3.3-1~debian.13~trixie
```

# 6 企业级做法（推荐）

对于 Docker 官方仓库：

不要手工：

```bash
apt download
reprepro includedeb
```

而是：

## 方案A

下载：

```bash
apt-get install \
    --download-only \
    docker-ce \
    docker-ce-cli \
    containerd.io \
    docker-buildx-plugin \
    docker-compose-plugin
```

然后：

```bash
reprepro -b /repo includedeb trixie *.deb
```

适合：

```text
GitHub Actions
离线仓库
```

---

## 方案B（更推荐）

直接使用 Aptly Mirror。

例如同步 Docker 官方仓库：

```bash
aptly mirror create \
    -architectures=amd64 \
    docker-trixie \
    https://download.docker.com/linux/debian \
    trixie \
    stable
```

更新：

```bash
aptly mirror update docker-trixie
```

然后创建快照：

```bash
aptly snapshot create \
    docker-20260709 \
    from mirror docker-trixie
```

发布：

```bash
aptly publish snapshot docker-20260709
```

这样：

```text
Docker官方仓库
↓
完整同步
↓
所有依赖
↓
版本保留
↓
快照
↓
离线仓库
```

完全类似：

```text
Rocky Linux

reposync
↓
createrepo
↓
nginx
```
