如果你的目标是：

```text
外网机器（GitHub Actions / 互联网服务器）
      ↓
下载 Debian 软件包及依赖
      ↓
U盘拷贝
      ↓
内网 Debian 安装
```

那么 **apt-offline** 确实是 Debian 官方支持的离线安装方案之一。

但是先说结论：

| 方案                         | 适合场景       | 推荐度   |
| -------------------------- | ---------- | ----- |
| apt-offline                | 少量服务器、补丁更新 | ⭐⭐⭐   |
| 本地APT仓库（dpkg-scanpackages） | 集群部署       | ⭐⭐⭐⭐⭐ |
| Aptly / reprepro           | 企业级离线仓库    | ⭐⭐⭐⭐⭐ |

对于你这种：

* Docker
* Kubernetes
* Harbor
* 多节点集群

实际上我更推荐 **APT本地仓库** 而不是 apt-offline。

因为 apt-offline 本质上是：

```text
内网机器生成需求
    ↓
外网机器下载
    ↓
回内网安装
```

更像：

```text
yumdownloader
dnf download
```

而不是：

```text
reposync
createrepo
```

# 一、apt-offline 工作原理

## 第一步：内网机器生成需求文件

假设内网 Debian 13：

安装 apt-offline：

```bash
apt install apt-offline
```

生成请求：

```bash
apt-offline set docker.sig \
    --install-packages \
    docker-ce,docker-ce-cli,containerd.io,docker-buildx-plugin,docker-compose-plugin
```

得到：

```text
docker.sig
```

里面记录：

```text
需要哪些包
需要哪些依赖
需要哪些元数据
```

---

# 二、把 docker.sig 带到外网机器

例如：

```text
U盘
docker.sig
```

---

# 三、外网机器下载

安装 apt-offline：

```bash
apt install apt-offline
```

下载：

```bash
apt-offline get docker.sig \
    --bundle docker.zip
```

结果：

```text
docker.zip
```

里面包含：

```text
Packages
Release
docker-ce*.deb
containerd*.deb
依赖包
```

---

# 四、拷回内网安装

```bash
apt-offline install docker.zip
```

APT数据库会更新。

然后：

```bash
apt install docker-ce
```

即可安装。

---

# 五、Docker 官方仓库如何支持 apt-offline

Docker 仓库必须提前配置：

```bash
/etc/apt/sources.list.d/docker.sources
```

例如：

```text
Types: deb
URIs: https://download.docker.com/linux/debian
Suites: trixie
Components: stable
Architectures: amd64
Signed-By: /etc/apt/keyrings/docker.asc
```

否则：

```bash
apt-offline set
```

不会知道 Docker 包来自哪里。

# 六、apt-offline 下载 Docker 的完整流程

## 内网

```bash
apt-offline set docker.sig \
  --update \
  --install-packages \
  docker-ce,docker-ce-cli,containerd.io,docker-buildx-plugin,docker-compose-plugin
```

---

## 外网

```bash
apt-offline get docker.sig \
  --bundle docker.zip
```

---

## 内网

```bash
apt-offline install docker.zip

apt install docker-ce
```

# 七、企业环境的问题

如果你有：

```text
20台 Debian
50台 Debian
100台 Debian
```

apt-offline 会变得很麻烦：

```text
机器A → A.sig
机器B → B.sig
机器C → C.sig
...
```

然后：

```text
A.zip
B.zip
C.zip
```

管理复杂。

---

# 八、企业最佳实践（推荐）

你已经在研究：

```text
GitHub Actions
Dockerfile
离线下载
```

实际上更适合直接生成：

```text
offline-repo/
├── pool/
├── Packages.gz
├── Release
└── install.sh
```

外网：

```bash
apt-get install --download-only
```

下载全部依赖。

生成：

```bash
dpkg-scanpackages . /dev/null > Packages
gzip -9 Packages
```

---

内网：

```bash
echo "deb [trusted=yes] file:/opt/offline-repo ./" \
> /etc/apt/sources.list.d/offline.list

apt update

apt install docker-ce
```

效果与联网完全一致。

---

# 九、如果一定要在 GitHub Actions 中使用 apt-offline

Dockerfile 需要额外安装：

```dockerfile
RUN apt-get update && \
    apt-get install -y apt-offline
```

然后：

```bash
apt-offline get docker.sig \
    --bundle /output/docker.zip
```

上传 Artifact：

```text
docker.zip
```

内网：

```bash
apt-offline install docker.zip
```

即可。

---

对于你的场景（长期维护 Docker、K8s、Harbor、Flink、Kafka 等内网集群），我建议直接走：

```text
GitHub Actions
      ↓
APT离线仓库
      ↓
U盘
      ↓
内网HTTP仓库
      ↓
apt install
```

这才是 Debian 世界里最接近 CentOS `reposync + createrepo` 的企业级方案。`apt-offline` 更适合补丁同步和单机离线更新，而不太适合作为整个集群的软件分发体系。
