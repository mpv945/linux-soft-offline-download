## 示例 SPEC（生产可用简化版）

```spec
Name:           nginx-lua
Version:        1.24.0
Release:        1%{?dist}
Summary:        Nginx with Lua module

License:        BSD
Source0:        nginx-1.24.0.tar.gz
Source1:        lua-nginx-module.tar.gz
Source2:        LuaJIT-2.1.0-beta3.tar.gz

BuildRequires:   gcc, make, pcre-devel, zlib-devel, openssl-devel

%description
Nginx compiled with Lua support (ngx_http_lua_module)

%prep
tar -xzf %{SOURCE0}
tar -xzf %{SOURCE1}
tar -xzf %{SOURCE2}

mv nginx-1.24.0 nginx
mv lua-nginx-module lua_module
mv LuaJIT-2.1.0-beta3 luajit

%build

# build LuaJIT
cd luajit
make && make install PREFIX=/usr/local/luajit
cd ..

export LUAJIT_LIB=/usr/local/luajit/lib
export LUAJIT_INC=/usr/local/luajit/include/luajit-2.1

cd nginx

./configure \
  --prefix=/etc/nginx \
  --sbin-path=/usr/sbin/nginx \
  --conf-path=/etc/nginx/nginx.conf \
  --error-log-path=/var/log/nginx/error.log \
  --http-log-path=/var/log/nginx/access.log \
  --with-http_ssl_module \
  --with-http_v2_module \
  --add-module=../lua_module

make -j$(nproc)

%install
make DESTDIR=%{buildroot} install

mkdir -p %{buildroot}/usr/lib/systemd/system

cat > %{buildroot}/usr/lib/systemd/system/nginx.service <<EOF
[Unit]
Description=nginx
After=network.target

[Service]
Type=forking
ExecStart=/usr/sbin/nginx
ExecReload=/usr/sbin/nginx -s reload
ExecStop=/usr/sbin/nginx -s quit

[Install]
WantedBy=multi-user.target
EOF

%files
/etc/nginx
/usr/sbin/nginx
/usr/lib/systemd/system/nginx.service
/var/log/nginx

%post
systemctl daemon-reload

%changelog
* Thu Jun 26 2026 you <you@example.com> - 1.24.0-1
- build nginx with lua module
```

# 二、SPEC 文件到底是什么？（核心理解）

SPEC = **RPM的“构建说明书”**

你可以把它理解成：

> 👉 Dockerfile（但不是构镜像，而是构 RPM）

---

## SPEC 的 6 个核心阶段

```text
1. 元信息（Name/Version）
2. 依赖（BuildRequires / Requires）
3. 源码准备（%prep）
4. 编译（%build）
5. 安装（%install）
6. 打包文件清单（%files）
```

---

## SPEC 执行流程图

```text
SOURCE → %prep → %build → %install → %files → RPM
```

# 三、最重要的 SPEC 结构（逐块讲清）

我用 nginx + lua 给你拆解：

---

## 1️⃣ 基础信息（RPM身份）

```spec
Name:           nginx-lua
Version:        1.24.0
Release:        1%{?dist}
Summary:        Nginx with Lua module

License:        BSD
```

### 含义：

| 字段      | 作用      |
| ------- | ------- |
| Name    | 包名      |
| Version | nginx版本 |
| Release | 第几次打包   |
| Summary | 描述      |

---

## 2️⃣ 构建依赖（编译时需要）

```spec
BuildRequires: gcc, make, pcre-devel, zlib-devel, openssl-devel
```

### 重点：

👉 这是“编译时依赖”，不是运行依赖

例如：

* nginx 编译需要 gcc
* https 需要 openssl-devel
* rewrite 需要 pcre-devel

---

## 3️⃣ 源码定义（Docker/CI会用到）

```spec
Source0: nginx-1.24.0.tar.gz
Source1: lua-nginx-module.tar.gz
Source2: LuaJIT-2.1.0-beta3.tar.gz
```

👉 这些文件来自：

```
rpmbuild/SOURCES/
```

---

## 4️⃣ 解压阶段（%prep）

```spec
%prep
tar -xzf %{SOURCE0}
tar -xzf %{SOURCE1}
tar -xzf %{SOURCE2}

mv nginx-1.24.0 nginx
mv lua-nginx-module lua_module
mv LuaJIT-2.1.0-beta3 luajit
```

### 作用：

👉 把源码准备成“可编译目录结构”

---

## 5️⃣ 编译阶段（%build）⭐最关键

```spec
%build

cd luajit
make && make install PREFIX=/usr/local/luajit
cd ..
```

---

### 设置环境变量（非常关键）

```spec
export LUAJIT_LIB=/usr/local/luajit/lib
export LUAJIT_INC=/usr/local/luajit/include/luajit-2.1
```

---

### nginx 编译核心

```spec
cd nginx

./configure \
  --prefix=/etc/nginx \
  --sbin-path=/usr/sbin/nginx \
  --conf-path=/etc/nginx/nginx.conf \
  --with-http_ssl_module \
  --with-http_v2_module \
  --add-module=../lua_module
```

---

### make

```spec
make -j$(nproc)
```

---

## 6️⃣ 安装阶段（%install）

👉 这里不是“安装到系统”，而是：

> 👉 安装到 RPM 构建目录（buildroot）

```spec
%install
make DESTDIR=%{buildroot} install
```

---

### systemd 文件（非常重要）

```spec
mkdir -p %{buildroot}/usr/lib/systemd/system
```

---

写 nginx.service：

```spec
cat > %{buildroot}/usr/lib/systemd/system/nginx.service <<EOF

ExecStart=/usr/sbin/nginx
ExecReload=/usr/sbin/nginx -s reload
ExecStop=/usr/sbin/nginx -s quit

[Install]
WantedBy=multi-user.target
EOF
```

---

## 7️⃣ 文件打包（%files）⭐决定RPM内容

```spec
%files

/usr/sbin/nginx
/usr/lib/systemd/system/nginx.service
/var/log/nginx
```

这里决定：

> 👉 RPM安装后“系统里会出现什么文件”

# ⚙️ 四、本地构建脚本（Docker内执行）

#!/bin/bash

# 🧩 二、关键：systemd 服务文件（核心）

## 📌 1. nginx.service（标准生产版）

在 SPEC 里写：

```spec id="s1m9ka"
%install
make DESTDIR=%{buildroot} install

# systemd 目录
mkdir -p %{buildroot}/usr/lib/systemd/system
mkdir -p %{buildroot}/var/run/nginx
mkdir -p %{buildroot}/var/log/nginx
mkdir -p %{buildroot}/etc/nginx/conf.d
```

---

## 📌 写 service 文件

```spec id="d8k3mz"
cat > %{buildroot}/usr/lib/systemd/system/nginx.service <<EOF
[Unit]
Description=High Performance Nginx (Lua Enabled)
After=network.target

[Service]
Type=forking

# 运行用户（必须）
User=nginx
Group=nginx

# PID 文件
PIDFile=/var/run/nginx/nginx.pid

# 启动
ExecStartPre=/usr/sbin/nginx -t
ExecStart=/usr/sbin/nginx
ExecReload=/usr/sbin/nginx -s reload
ExecStop=/usr/sbin/nginx -s quit

# 重载信号
KillSignal=SIGQUIT

# 运行目录
WorkingDirectory=/etc/nginx

# 防止崩溃限制
LimitNOFILE=65535

# 重启策略
Restart=on-failure
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF
```

# 👤 三、自动创建 nginx 用户（非常关键）

在 SPEC 添加：

```spec id="p0k2ls"
%pre
getent group nginx >/dev/null || groupadd -r nginx
getent passwd nginx >/dev/null || useradd -r -g nginx -s /sbin/nologin nginx
exit 0
```
