# 一、整体认知（先建立概念）

Debian 打包本质是：

> ❗ “把源码 + 编译规则 + 元数据 → 变成标准 .deb 包”

核心三件套：

| 文件             | 作用                  |
| -------------- | ------------------- |
| debian/control | 包的“说明书 + 依赖关系”      |
| debian/rules   | “怎么编译（核心脚本）”        |
| debian/compat  | “使用 debhelper 版本规则” |

# 二、debian/control（最重要的“包说明书”）

## 📌 它的作用

👉 类似：

* Maven pom.xml
* pip requirements + metadata
* RPM spec 的 header

它定义：

* 包叫什么
* 依赖什么
* 安装什么
* 运行环境要求

## 📌 参数逐个讲清楚

---

### 1️⃣ Source

```text
Source: nginx-lua
```

👉 源码包名称（不是最终安装包）

作用：

* 标识“这个源码项目叫什么”
* 一个 Source 可以生成多个 Package

---

### 2️⃣ Section

```text
Section: web
```

👉 分类

常见值：

| 值     | 说明    |
| ----- | ----- |
| web   | web服务 |
| utils | 工具    |
| libs  | 库     |
| admin | 管理工具  |

👉 影响 apt 分类展示

---

### 3️⃣ Priority

```text
Priority: optional
```

👉 包重要程度

| 值        | 含义          |
| -------- | ----------- |
| required | 系统必须        |
| standard | 标准系统        |
| optional | 可选（你基本都用这个） |

---

### 4️⃣ Build-Depends（非常关键）

```text
Build-Depends: debhelper-compat (= 13),
               libpcre3-dev,
               zlib1g-dev,
               libssl-dev,
               libluajit-5.1-dev
```

👉 **编译依赖**

意思是：

> ❗ “在构建 .deb 时必须先安装这些依赖”

---

📌 nginx + lua 常见依赖解释：

| 包            | 作用            |
| ------------ | ------------- |
| debhelper    | Debian打包工具    |
| libpcre3-dev | nginx rewrite |
| zlib1g-dev   | gzip压缩        |
| libssl-dev   | HTTPS         |
| libluajit    | Lua运行环境       |

---

### 5️⃣ Package

```text
Package: nginx-lua
```

👉 最终安装包名字：

```bash
apt install nginx-lua
```

---

### 6️⃣ Architecture

```text
Architecture: amd64
```

👉 支持架构：

| 值     | 含义       |
| ----- | -------- |
| amd64 | 64位Linux |
| arm64 | ARM服务器   |
| all   | 脚本类      |

---

### 7️⃣ Depends（运行依赖）

```text
Depends: ${shlibs:Depends}, ${misc:Depends}
```

👉 运行时依赖

解释：

| 写法                | 作用            |
| ----------------- | ------------- |
| ${shlibs:Depends} | 自动分析动态库       |
| ${misc:Depends}   | debhelper自动补充 |

---

📌 重点：

👉 你一般**不用手写依赖**
👉 交给 dpkg 自动分析

---

### 8️⃣ Description

```text
Description: Nginx with Lua support
 Custom nginx build with lua-nginx-module
```

👉 包说明

规则：

* 第一行 = 简短描述
* 第二行 = 详细描述（必须空格开头）

# 三、debian/rules（核心编译脚本）

## 📌 它是什么？

👉 本质：Makefile

控制：

> ❗ “如何 configure / make / install”

---

## 📌 标准结构

```makefile
#!/usr/bin/make -f

%:
	dh $@
```

👉 这叫：

> debhelper 自动模式

---

## 📌 nginx 这种项目必须 override

因为 nginx 不是标准 autotools 项目

---

## 📌 完整示例（你用的）

```makefile
override_dh_auto_configure:
	./configure \
		--prefix=/usr/share/nginx \
		--sbin-path=/usr/sbin/nginx \
		--conf-path=/etc/nginx/nginx.conf \
		--with-http_ssl_module \
		--with-http_v2_module \
		--add-module=../lua-nginx-module
```

---

## 📌 逐个解释

---

### 1️⃣ override_dh_auto_configure

👉 覆盖默认 configure 行为

默认 Debian 会：

```bash
./configure
```

但 nginx 需要：

```bash
./configure + 参数
```

---

### 2️⃣ --prefix

```bash
--prefix=/usr/share/nginx
```

👉 nginx 安装路径

| 路径               | 含义    |
| ---------------- | ----- |
| /usr/sbin/nginx  | 可执行文件 |
| /etc/nginx       | 配置    |
| /usr/share/nginx | 资源    |

---

### 3️⃣ --sbin-path

```bash
--sbin-path=/usr/sbin/nginx
```

👉 nginx 主程序位置

---

### 4️⃣ --conf-path

```bash
--conf-path=/etc/nginx/nginx.conf
```

👉 配置文件路径（很关键）

---

### 5️⃣ --with-http_ssl_module

👉 开启 HTTPS

---

### 6️⃣ --add-module

```bash
--add-module=../lua-nginx-module
```

👉 ⭐ Lua模块关键点

说明：

> 把外部模块“编译进 nginx”

---

## 📌 override_dh_auto_build

```makefile
override_dh_auto_build:
	make -j$(nproc)
```

👉 并行编译

---

## 📌 override_dh_auto_install

```makefile
override_dh_auto_install:
	make install DESTDIR=$(CURDIR)/debian/nginx-lua
```

👉 重点！！！

### DESTDIR 是什么？

👉 “假安装目录”

不会装到系统，而是：

```text
debian/nginx-lua/
```

然后 deb 打包工具再封装成 .deb
